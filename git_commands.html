Basic git for beginners

Introduce git

Go to directory that you want to control with git and write command:

1-git init

Now git directory created in you current directory and git start :)
To chack what is you git status write:

2-git status

After see status you should see some "Untracked files" and some file that git say "Changes not staged for commit" that zsh show them in red color. to add file to git write:

3-"git add <file>"

To add all file use:

3.5-git add -A

Now all of your files are in stage state so ready to commit But!!!!! befor do it you should Introduce your self to git and say who you are:

3.75-git config --global user.name "mohammadarastenia"

and set you email:

3.85-git config --global user.email mohammadarastenia@yahoo.com

Now you can commit your file to git

4-git commit -m 'git first commit :)'

Now git status show nothing file because all of them committed. but if you change you file first status become red and after add it it become green to you make new commit :)

5-git log

Show all your commit to quit log press ‘q’

6-git diff HEAD

Show difference between HEAD(your last commit) of a file with last changing.

7-git diff --staged

Show difference between staged (git add …) file that still not committed. If you are not like changes you can pick up you file from stage:

8-"git reset <file>"

But if you really don’t like all changes you can go to last commit ofyour file.

9-"git checkout -- <file>"

To remove a file from git type:

10-"git rm <file>"

Git branch

To know witch branch you are working on it write:

1-git branch

This command show all branch and your working branch highlight and has ‘*’ nearby. To Create new branch write:

2-"git branch<new_branch_name>"

Now you are creating you new branch but still working on old (master) branch. To switch in the new branch write:

3-"git checkout <new_branch_name>"

Now you are ready to improve your branch as a copy of master branch. After complete your work, save(git add) it and make a commit for it. first go to master (git checkout master) and you see all of your work vanish but don’t worry. Just type:

4- "git merge <new_branch_name>"

To remove a branch from git Type:

5-"git branch -d <new_branch_name>"

Git on the Server/git remote

The first step is to download a project repository from github by using command:

1-"git clone <URL>"

Please don't make extra directory because git clone download all file in separate file.so don't worry. After download you can change and save it after that commit them. but now if you see git status, it says your branch is ahead of orgin/master by 1 commit (orgin is name of remote). So next step is to Upload you change to github.

2-git push orgin master

After this step github authenticate you to know that who you are and have permition to change and commit. If all things are right you can see you changes on github. and number of commit increased by your push.

Now the problem is if some one change orginal repository on github, how do you update your local master to last update. The answer is:

3-git pull orgin master

Now forget all above commands ,assume you are in local host and your systwm admin want you to work on a repository remotly. first step is to define your remote for git as an orgin by using:

1.1-git remote add orgin <URL of your repo>

Now following command say you what is your remote

1.2-git remote

After complete your work. following command upload your work, if every things is true.

2.1-git push -u orgin master
